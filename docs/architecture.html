Compact + scatter
Problem: A single consultation dumps ~44K tokens of tool output into the main context (list_concepts ~12.5K, get_subgraph ~26.6K, ask_book ~5K), causing context exhaustion and poor pruning. The fix: server-side compaction (smaller tool outputs) + scatter-gather (parallel subagents for graph analysis).

The raw ~5K token graph data in each subagent's context is discarded after the summary is returned — only ~300 tokens per subagent enter the main context.

Book Pattern Grounding
Supervisor Architecture
Ch 5
The main Claude context acts as supervisor, orchestrating the consultation workflow, dispatching subagents, and merging their results into a coherent recommendation.
Agent Delegates to Agent
Ch 8 / Ch 14
Each seed concept is explored by a dedicated subagent that calls get_subgraph independently, analyzes relationships, and returns a compact summary to the supervisor.
Pass-by-Reference
Ch 7
Tool outputs stay compact by default (IDs, names, categories). Full data (definitions, descriptions) is retrieved only when explicitly requested via opt-in parameters.
Shared Epistemic Memory
Ch 9
The knowledge graph in MotherDuck serves as shared memory that all subagents can query independently without duplicating data in any single context window.
